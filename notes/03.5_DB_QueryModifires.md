# Mongoose: Model Methods & Query Modifiers

**Model Methods are the starters.** You call them directly on your capitalized Model (e.g., `User.find()`). They initiate the interaction with the database.
Because `.aggregate()` is a Model Method, it starts its own process. You cannot chain Query Modifiers like `.select()` or `.populate()` onto an `.aggregate()` call. You have to do all your shaping and populating inside the aggregation pipeline itself.

**Query Modifiers are the shapers.** You chain them after a Model Method (e.g., `.select()` or `.sort()`) to shape the data before it comes back.
You chain these onto the "Read" Model Methods (like `find()` or `findById()`). They don't touch the database on their own; they just give Mongoose extra instructions on how to format the data before handing it to you.

---

## 1. `select(fields)`

Controls which fields are returned or hidden.

In database terminology, this is called **projection**. It allows you to specify exactly which fields you want MongoDB to return, or which fields you want to hide.

Whenever you don't need the entire document. It speeds up your database queries, reduces memory usage on your server, and hides sensitive data (like passwords or reset tokens) from reaching the frontend.

- **Arguments:** It takes either a space-separated string or an object. You use `1` (or include the name) to include it, and `0` (or `-` in strings) to exclude it.

**1. String Syntax (Most Common)**

```javascript
// Get ONLY name and email. By default, _id is ALWAYS returned.
await User.find().select("name email");

// Get everything EXCEPT the password and version key (__v)
await User.find().select("-password -__v");
```

**2. Object Syntax (More structured)**

```javascript
// Get name, exclude _id
await User.find().select({name: 1, _id: 0});
```

## 2. `populate(path)`

MongoDB is a NoSQL database, meaning it doesn't have traditional SQL JOINs. Instead, we save references (ObjectIds) in documents. `.populate()` is Mongoose's way of automatically replacing those ObjectIds with the actual, full documents from another collection.

When your schema has an ObjectId referencing another model (e.g., a Ticket has a `user` field), and you want to fetch the user's name and email without writing a second query.

- **Arguments:** A string (the path to populate), or a configuration object for advanced control.

**Basic Population:**

```javascript
// Replace the 'user' ObjectId with the full User document
await Ticket.find().populate("user");
```

**Advanced Population (The Object Syntax):**

```javascript
await Ticket.find().populate({
  path: "user", // The field to populate
  select: "name email -_id", // Only get the user's name and email
});
```

**Deep Population:**
What if your Event has an array of tickets, and each Ticket has a user? You can nest populates!

```javascript
await Event.findById(eventId).populate({
  path: "tickets",
  populate: {
    path: "user",
    select: "name",
  },
});
```

## The Pagination Trio: `.sort()`, `.skip()`, and `.limit()`

- **`.sort()`**: Orders your data. `{ createdAt: -1 }` means descending (newest first). `{ title: 1 }` means ascending (A-Z).
- **`.limit()`**: Caps the number of documents returned. `limit(10)` returns exactly 10 items.
- **`.skip()`**: Skips a specific number of documents before starting to return them.

```javascript
const page = 2;
const itemsPerPage = 10;

const events = await Event.find()
  .sort({date: 1}) // 1. Order by date (soonest first)
  .skip((page - 1) * itemsPerPage) // 2. Skip the first 10 (Page 1)
  .limit(itemsPerPage) // 3. Grab the next 10 (Page 2)
  .select("title date"); // 4. Only grab title and date
```
