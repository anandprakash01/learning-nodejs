# MongoDB Operators

Query Operators are not strictly interchangeable with aggregation, and no, you cannot easily swap `$sum` for `$inc` in a standard update.

To remember, separate MongoDB operators into three distinct buckets:

- **Query Operators (The Bouncers):** Used in `find()` or the first part of an update. They filter data. (e.g., `$gt`, `$in`, `$or`). ("Only let documents through if age > 18").
- **Update Operators (The Mechanics):** Used in the second part of `findOneAndUpdate()`. They permanently change data in the database. (e.g., `$inc`, `$set`, `$push`). ("Add 1 to the capacity").
- **Aggregation Operators (The Accountants):** Used in `.aggregate()`. They do math and reshape data on the fly, but don't permanently change the database. (e.g., `$sum`, `$avg`).

_(Note: In newer versions of MongoDB, you actually can use aggregation pipelines inside updates, which blurs the lines a bit, but for standard CRUD operations, keep them separate in your mind!)_

## Query Operators

### 1. Comparison Operators (The most common)

- **`$eq` (Equal To):** Matches values that are equal. (Usually implicit, you rarely type this out).

  ```javascript
  Event.find({status: "Published"}); // Implicit (Standard)
  Event.find({status: {$eq: "Published"}}); // Explicit
  ```

- **`$ne` (Not Equal):** Matches all values that are NOT equal to a specified value.

  ```javascript
  Event.find({status: {$ne: "Cancelled"}});
  ```

- **`$gt` / `$gte` (Greater Than / Greater Than or Equal):**

  ```javascript
  // Find events with strictly more than 100 capacity
  Event.find({capacity: {$gt: 100}});
  ```

- **`$lt` / `$lte` (Less Than / Less Than or Equal):**

  ```javascript
  // Find tickets cheaper than or equal to $50
  Ticket.find({price: {$lte: 50}});
  ```

- **`$in` (In Array):** Matches any of the values specified in an array. (Extremely useful!)

  ```javascript
  // Find events located in either NY, LA, or Chicago
  Event.find({city: {$in: ["NY", "LA", "Chicago"]}});
  ```

- **`$nin` (Not In Array):** Matches none of the values specified in an array.
  ```javascript
  Event.find({city: {$nin: ["Remote", "Online"]}});
  ```

### 2. Logical Operators

These combine multiple rules together.

- **`$or`:** Joins query clauses with a logical OR. Returns documents that match at least one clause.

  ```javascript
  // Find events that are either Free OR have a VIP option
  Event.find({
    $or: [{price: 0}, {hasVIP: true}],
  });
  ```

- **`$and`:** Joins query clauses with a logical AND. (Usually implicit, but useful when filtering the same field twice).
  ```javascript
  // Find events priced between 50 and 100
  Event.find({
    $and: [{price: {$gte: 50}}, {price: {$lte: 100}}],
  });
  ```

### 3. Element Operators

These check the structure of the document itself.

- **`$exists`:** Checks if a field exists in the document, even if it's null.
  ```javascript
  // Find events that have a 'discountCode' field attached to them
  Event.find({discountCode: {$exists: true}});
  ```

### 4. Evaluation Operators

These are the complex, advanced filters.

- **`$regex`:** Allows you to search strings using Regular Expressions (like "contains" or "starts with").

  ```javascript
  // Find events where the title contains "Tech" (case-insensitive)
  Event.find({title: {$regex: /tech/i}});
  ```

- **`$expr`:** Allows you to compare two different fields within the same document against each other.
  ```javascript
  // Find events where attendeesCount is exactly equal to maxCapacity
  Event.find({$expr: {$eq: ["$attendeesCount", "$maxCapacity"]}});
  ```

### 5. Array Operators

These filter documents based on arrays stored inside them.

- **`$size`:** Matches arrays with a specific length.

  ```javascript
  // Find events that have exactly 5 tags
  Event.find({tags: {$size: 5}});
  ```

- **`$all`:** Matches arrays that contain all elements specified in the query.
  ```javascript
  // Find events that have BOTH "Music" and "Outdoor" in their tags array
  Event.find({tags: {$all: ["Music", "Outdoor"]}});
  ```

## Update Operators

These are categorized by what part of the document they modify.

### 1. Field Operators (Modifying basic values)

- **`$set`:** Replaces the value of a field with a new value. (If the field doesn't exist, it creates it).

  ```javascript
  // Change the event status to "Sold Out"
  await Event.findByIdAndUpdate(eventId, {$set: {status: "Sold Out"}});
  ```

- **`$unset`:** Completely deletes a field from a document. (You pass 1 or "" to trigger it).

  ```javascript
  // Remove the discountCode field from this event entirely
  await Event.findByIdAndUpdate(eventId, {$unset: {discountCode: 1}});
  ```

- **`$inc`:** Increments (adds to) or decrements (subtracts from) a number field.

  ```javascript
  // Add 1 to attendeesCount, and subtract 1 from availableTickets
  await Event.findByIdAndUpdate(eventId, {
    $inc: {attendeesCount: 1, availableTickets: -1},
  });
  ```

- **`$mul`:** Multiplies the current value of a number field.

  ```javascript
  // Surge pricing: Multiply the current price by 1.5
  await Event.findByIdAndUpdate(eventId, {$mul: {price: 1.5}});
  ```

- **`$rename`:** Changes the name of the key/field itself.

  ```javascript
  // Rename 'attendeesCount' to 'totalRSVPs' in the database
  await Event.updateMany({}, {$rename: {attendeesCount: "totalRSVPs"}});
  ```

- **`$min` / `$max`:** Only updates the field if the new value is lower (`$min`) or higher (`$max`) than the current value.
  ```javascript
  // Only update the 'highestScore' if the user's new score of 2500 is > their current score
  await User.findByIdAndUpdate(userId, {$max: {highestScore: 2500}});
  ```

### 2. Array Operators (Modifying lists of data)

If your document has an array (like an array of tags or an array of userIds), you must use these to manipulate the array safely.

- **`$push`:** Appends an item to the end of an array.

  ```javascript
  // Add "Music" to the tags array
  await Event.findByIdAndUpdate(eventId, {$push: {tags: "Music"}});
  ```

- **`$addToSet`:** Exactly like `$push`, but prevents duplicates. If "Music" is already in the tags array, it does nothing.

  ```javascript
  // Add the user to the likedBy array, but only if they haven't liked it already
  await Event.findByIdAndUpdate(eventId, {$addToSet: {likedBy: userId}});
  ```

- **`$pop`:** Removes the first or last item from an array. (Use 1 for the last item, -1 for the first).

  ```javascript
  // Remove the very last tag in the array
  await Event.findByIdAndUpdate(eventId, {$pop: {tags: 1}});
  ```

- **`$pull`:** Removes specific items from an array that match a condition.
  ```javascript
  // Remove "userId" from the attendees array (e.g., user canceled ticket)
  await Event.findByIdAndUpdate(eventId, {$pull: {attendees: userId}});
  ```

### 3. Array Modifiers (The "Secret Weapons")

These don't work on their own. You use them inside a `$push` to do advanced array manipulations.

- **`$each`:** Pushes multiple items at once instead of just one.

  ```javascript
  // Add THREE tags at the exact same time
  await Event.findByIdAndUpdate(eventId, {
    $push: {tags: {$each: ["Music", "Live", "Outdoor"]}},
  });
  ```

- **`$slice`:** Caps the size of the array. Perfect for keeping a "Recent Searches" or "Last 5 Logins" list without the array growing infinitely!
  ```javascript
  // Push the new login date, but only keep the last 5 dates in the array
  await User.findByIdAndUpdate(userId, {
    $push: {
      loginHistory: {
        $each: [new Date()],
        $slice: -5, // Keep only the last 5 items
      },
    },
  });
  ```

## Aggregation Operators

There are actually two different types of Aggregation Operators:

- **Pipeline Stages:** These are the actual "stations" on the assembly line (e.g., `$match`, `$group`).
- **Expression Operators:** These are the specific math or formatting tools you use inside those stations (e.g., `$sum`, `$avg`, `$concat`).

### Part 1: Pipeline Stages (The Assembly Line Stations)

You pass these as objects inside your `.aggregate([ ... ])` array. The order you place them in matters immensely, because the output of one stage becomes the input of the next.

**The Core Stages:**

- **`$match`:** Filters the documents. It acts exactly like a standard `.find()`. You almost always want this to be your very first stage so you aren't doing heavy math on documents you don't need.

  ```javascript
  // Only pass "VIP" tickets down the assembly line
  {
    $match: {
      ticketType: "VIP";
    }
  }
  ```

- **`$group`:** The most powerful stage. It bundles documents together based on a specific field and allows you to calculate totals, averages, etc., for that bundle.

  ```javascript
  // Bundle by eventId, and count how many tickets are in each bundle
  {
    $group: {
      _id: "$eventId", // _id is required here. It tells MongoDB WHAT to group by.
      totalTickets: { $sum: 1 }
    }
  }
  ```

- **`$project`:** Reshapes the documents. It acts like `.select()`, but much stronger. You can hide/show fields, rename them, or create entirely new fields based on math.

  ```javascript
  // Hide the _id, show the title, and create a new field called "isSoldOut"
  {
    $project: {
      _id: 0,
      title: 1,
      isSoldOut: { $eq: ["$attendeesCount", "$capacity"] }
    }
  }
  ```

- **`$sort`, `$skip`, `$limit`:** These do exactly what their Query Modifier counterparts do, but inside the pipeline.
  ```javascript
  { $sort: { totalRevenue: -1 } }, // Sort highest revenue first
  { $skip: 10 },
  { $limit: 10 }
  ```

**The Advanced Stages:**

- **`$lookup`:** This is MongoDB's version of a SQL JOIN or a Mongoose `.populate()`. It pulls in data from another collection. (It is generally faster than `.populate()`).

  ```javascript
  // Look inside the 'users' collection and attach the user data to this ticket
  {
    $lookup: {
      from: "users",           // The actual collection name in MongoDB
      localField: "userId",    // The field in the current Ticket document
      foreignField: "_id",     // The field in the User document to match against
      as: "userDetails"        // The name of the new array where the data will be placed
    }
  }
  ```

- **`$unwind`:** Deconstructs an array. If a document has an array of 3 tags, `$unwind` will destroy the original document and replace it with 3 identical documents, each containing just one of those tags. It is essential for grouping by array elements.
  ```javascript
  // Turns { title: "Fest", tags: ["Music", "Food"] }
  // Into { title: "Fest", tags: "Music" } AND { title: "Fest", tags: "Food" }
  {
    $unwind: "$tags";
  }
  ```

### Part 2: Expression Operators (The Calculators)

You use these inside your `$group` or `$project` stages to actually manipulate the data.

**Accumulators (Used mostly in `$group`):**

- **`$sum`:** Adds numbers together. (Or counts documents if you do `$sum: 1`).

  ```javascript
  // Adds up the 'price' field of all documents in the group
  totalRevenue: {
    $sum: "$price";
  }
  ```

- **`$avg`:** Calculates the average of all numbers in the group.

  ```javascript
  averageAge: {
    $avg: "$age";
  }
  ```

- **`$min` / `$max`:** Finds the lowest or highest value in the group.

  ```javascript
  cheapestTicket: {
    $min: "$price";
  }
  ```

- **`$push`:** (Yes, it exists here too!) Gathers a specific field from all documents in the group and creates a new array of those values.
  ```javascript
  // Creates a list of all user IDs who bought this type of ticket
  allBuyers: {
    $push: "$userId";
  }
  ```

**String & Date Formatters (Used mostly in `$project`):**

- **`$concat`:** Joins multiple strings together.

  ```javascript
  // Combines first and last name into one string
  fullName: {
    $concat: ["$firstName", " ", "$lastName"];
  }
  ```

- **`$year` / `$month` / `$dayOfMonth`:** Extracts specific parts of a Date object. This is amazing for generating monthly reports.
  ```javascript
  // Extracts the month (1-12) from the 'createdAt' timestamp
  registrationMonth: {
    $month: "$createdAt";
  }
  ```

## `$expr` and The Race Condition Trap

`$expr` stands for "Expression.
It unlocks the power of the Aggregation Pipeline right inside a normal `.find()` or `.findOneAndUpdate()` query. It allows you to say: "Compare Field A to Field B inside this exact document."

Using `$inc` like this is the absolute best practice for counting things in MongoDB.

### Why it Prevents Race Conditions:

To understand this, understand the "Read-Modify-Write" trap.
Imagine you are selling tickets, and two users (User A and User B) click "Buy Ticket" at the exact same millisecond. The event currently has 10 attendees.

**1. Fetch the document (Both requests get attendeesCount = 10)**

```javascript
const event = await Event.findById(eventId);
```

**2. Modify in JavaScript (Both calculate 10 + 1 = 11)**

```javascript
event.attendeesCount += 1;
```

**3. Write back to database (Both save 11 to the database)**

```javascript
await event.save();
```

Even though two people bought tickets, the count only went up by 1! This is a race condition.

**The GOOD Way (The `$inc` approach):**

```javascript
await Event.findByIdAndUpdate(eventId, {$inc: {attendeesCount: 1}});
```

This is called an **Atomic Operation**. You bypass JavaScript logic entirely and send a direct command to MongoDB: "I don't care what the current number is, just add 1 to it."

MongoDB handles this internally. It locks the document for a fraction of a millisecond, adds 1 for User A, unlocks it, locks it again, adds 1 for User B, and unlocks it. The count safely becomes 2.

`$inc` is completely different from Aggregation.
`$inc` here is an Update Operator. It permanently changes data stored in your database.
Aggregation operators (like `$sum` or `$add`) are Processors. They calculate numbers on the fly to return a result to you, but they do not change the actual data saved in your database.

## Powerful Update Operators

_(Note: These are additional examples of the Update Operators defined above)._

MongoDB gives you a whole toolkit of these Update Operators to modify data directly in the database without vulnerable JavaScript math.

### The Array Operators (Extremely Useful)

If your Event model stores an array of user IDs instead of just a number count, you use these:

- **`$push`:** Appends an item to an array.

  ```javascript
  // Adds the userId to the end of the attendees array
  await Event.findByIdAndUpdate(eventId, {$push: {attendees: userId}});
  ```

- **`$addToSet`:** Acts exactly like `$push`, but prevents duplicates. If the user is already in the array, it does nothing. (Perfect for "Like" buttons or RSVP lists).

  ```javascript
  await Event.findByIdAndUpdate(eventId, {$addToSet: {attendees: userId}});
  ```

- **`$pull`:** Removes an item from an array.
  ```javascript
  // Removes this specific userId from the attendees array (e.g., User cancels ticket)
  await Event.findByIdAndUpdate(eventId, {$pull: {attendees: userId}});
  ```

### The Field Operators

- **`$set`:** Changes a specific field to a specific value. (This is what Mongoose does behind the scenes normally).

  ```javascript
  await Event.findByIdAndUpdate(eventId, {$set: {status: "Sold Out"}});
  ```

- **`$unset`:** Completely deletes a field from the document.

  ```javascript
  // Removes the 'discountCode' field entirely from this event
  await Event.findByIdAndUpdate(eventId, {$unset: {discountCode: ""}});
  ```

- **`$mul`:** Multiplies a field's current value.

  ```javascript
  // Doubles the current price
  await Event.findByIdAndUpdate(eventId, {$mul: {price: 2}});
  ```

## To Avoid Race Condition

Find this event, but ONLY if `attendeesCount` is less than `capacity`. If found, increment the count by 1.

```javascript
const event = await Event.findOneAndUpdate(
  {
    _id: eventId,
    $expr: {$lt: ["$attendeesCount", "$capacity"]},
  },
  {
    $inc: {attendeesCount: 1},
  },
  {new: true}, // Returns the updated document
);
```
