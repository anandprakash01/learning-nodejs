➖ process is a massive global object built directly into Node.js. It represents the actual C++ program (Node) running on your Operating System (Windows/Mac/Linux).
process.env is a specific sub-object. "Env" stands for Environment Variables.

When Node.js boots, Operating System allocates a block of RAM for the Node C++ engine.
The Operating System takes a snapshot of its own global variables (like your computer's username, your OS version, and your terminal paths) and injects them into that block of RAM.
Node.js exposes that RAM to your JavaScript code via the process.env object. the OS injects environment variables as raw strings into the C++ memory space at `process.env`. 

➖➖➖ dotenv
"dotenv" is a "File System to RAM" injector. It allows you to write your secrets in a physical text file (.env), and it mimics the Operating System by injecting those secrets into the live process.env memory space.
import 'dotenv/config', triggering an automatic script inside the dotenv package.
behind the scenes:
=> File System Read: It uses Node's native fs (File System) module to scan your hard drive for a file literally named .env.
=> String Parsing: It reads the entire file as one giant text string. It splits the string at every new line. Then, it splits each line at the = sign.
=> The Mutation: It takes the left side of the = (the key) and the right side (the value) and literally executes a memory mutation: process.env[key] = value.

If successful: It returns { parsed: { PORT: '3000', MONGO_URI: 'mongodb://...' } }
If it fails (no file found): It returns { error: Error('ENOENT: no such file') }

The "Mistake": 
Amateurs scatter `process.env.PORT` or `process.env.JWT_SECRET` throughout hundreds of files. If a DevOps engineer forgets to set a variable in the production container, the server boots normally, but crashes days later when a specific line of code tries to access the missing secret. This is a tracing nightmare.

Now that `dotenv` has injected our .env file into process.env, we have a massive security and stability problem. Because process.env ONLY understands strings, process.env.PORT is the text string "3000".
